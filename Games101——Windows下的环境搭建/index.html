# C/C++ 环境配置

VSCode 中 C/C++ 环境的配置网上一堆教程，我最开始配置的时候看了看 B 站和 CSDN 的配置教程，但是总是达不到我满意的效果 (；′⌒`)，直到有一天我看了看 VSCode 文档，马上就配置出了我想要的效果，果然看文档才是最好的 φ(゜▽゜ *)♪

# 创建工作区

  • 在安装好了编译器和 VSCode 的 C/C++ 插件的前提下，打开 VSCode，新建一个文件夹在 VSCode 下打开

这里值得注意一下 VSCode 的工作区的概念，VSCode 默认将每个打开的文件夹视为工作区

工作区根目录下会生成一个 .vscode 文件夹，这个文件夹默认是隐藏的，可通过打开 文件资源管理器 的 显示隐藏文件 来将其显示

.vscode 文件夹下的 json 文件中写入了对于工作区的配置，该配置可参照 VSCode 的文档进行自定义，每个工作区都可做不同的配置，搭配丰富的扩展，VSCode 的功能可以变得非常强大，所以 VSCode 就是 yyds！

}

  • 在终端打开该文件，若是运行成功，则 C/C++ 的构建已配置成功

# 提一嘴 VSCode 中 CMake 的使用

Ctrl+Shift+p 打开 VSCode 的指令面板，然后输入 cmake:q ，VSCode 会根据输入自动提示，然后选择 CMake: Quick Start

然后选择适合你的编译器（VSCode 会将你安装的编译器都罗列出来供你选择)

然后根据提示输入项目名称，选择该项目是创建库还 S 是可执行文件，之后 VSCode 就会为你生成一个 CMake 项目
