树莓派搭建wordpress博客教程

掌控-物质 · 2020-8-24 · 次阅读

自己一直有做网站，之前用的是hexo搭建静态博客，无奈发现限制太大，所以想尝试下动态博客，然而网上的教程都要租云服务器，对于我这种常年白嫖的穷鬼来说怎么顶得住，碰巧手头有个树莓派，放着也是浪费，就拿来用用吧~

另外提一句，本教程中除了vnc连接的操作都可以在基于linux的服务器平台使用，据说宝塔面板喜提0day漏洞，所以这个操作应该也能帮到不少人

可行性分析：

  • 树莓派的官方系统是基于linux的魔改版debian，所以安装服务器所需配件理论上没有问题
  • 由于我采用的是树莓派3b+，所以性能方面也无问题
  • 值得一提的是，我们本地的服务器需要做内网穿透才可以进行外部访问，虽说我有公网ip，但总不能让访问的人把我家宽带占满然后我无法用网吧······
  • 所以继续白嫖，一通寻找，找到了可以白嫖的内网穿透：sakura frp
  • 然后是域名，虽然我依然想白嫖，但无奈sakura frp把能白嫖的域名全封禁了，那就只好去腾讯云买个便宜的域名了（悲）
  • ps：记得域名要备案

开始操作

安装树莓派系统

所需配件：

  • 树莓派一块（建议选择3b以上，不然性能不够）
  • 一个容量大于10G的SD卡（装系统不需要这么多，但是后期考虑到博客需要存储的东西，可以适当扩充点）
  • 树莓派官方系统镜像
  • 树莓派官方系统烧录软件
  • 读卡器
  • 充电器与充电线（供电用）

开始安装

把你的SD卡插入读卡器，读卡器插入电脑，打开官方烧录软件

在CHOOSE OS中选择你下载的系统镜像（如果是压缩包记得解压，最终是img格式），在CHOOSE SD CARD中选择你插入的SD卡，点击WRITE

烧录完成后，你的SD卡电脑会无法读取，这时不用慌，读卡器拔了再插回去就好了

设置SSH和WIFI

由于我们的树莓派没有屏幕，所以我们需要让电脑连接，连接需要SSH，但树莓派的SSH默认是关闭的，所以我们需要打开，而树莓派连接到我们的局域网需要用WIFI，如果你有多余的没用的网线，用网线连接也可以

设置SSH

电脑打开SD卡，在根目录下新建文本文件，重命名为ssh

记得把后缀删掉
记得把后缀删掉
记得把后缀删掉

重要的事情说三遍

设置WIFI

SD卡根目录下继续新建文本文件，重命名为wpa_supplicant.conf

同样.txt的后缀要删掉

然后打开这个文件，在里面输入如下内容

country=CN
ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
update_config=1
network={
ssid="你的无线网络名称"
key_mgmt=WPA-PSK
psk="你的wifi密码"
}

无线网络名称和wifi密码请填写自己的

保存，关闭，然后拔出SD卡，插入树莓派，你的系统已经安装完成了

连接到你的树莓派

连接到树莓派有两种方式，我推荐两种方式都用一下，因为操作的时候两种方式各有各的用处

所需软件：

  • putty（我的版本比较老，教程里的界面可能和你们的不一样）
  • VNC

putty连接

打开你家路由器的设置，一般你家路由器的地址是192.168.1.1或192.168.3.1，如果都不是，请看你的路由器说明书

找到树莓派的ip地址，将其设置为固定ip（这个操作应路由器而异，恕我无法讲解）

打开putty

如图，在主机名称处输入你的树莓派ip，连接类型选择SSH，点击打开即可

然后可能会弹出什么安全隐患，点确定即可

然后是一个登陆过程，树莓派默认账户名是pi，密码是raspberry，输入登陆即可

看到这样的提示就说明成功了

VNC连接

用VNC连接需要打开树莓派的VNC开关

在putty中输入sudo raspi-config，跳出如下界面

选择第五个，回车，选择VNC，将其打开即可

然后打开你的VNC软件，新建一个会话，输入树莓派的ip，名称随便起，然后连接后会让你输入用户名和密码，就是上面树莓派的默认名称和密码

连接上之后按照引导进行设置即可

然后树莓派会重启，输入你重新设置的密码连接即可

安装和配置服务器所需

网上很多教程都说要换源，但是搭建服务器推荐不换源，不然问题会很多很多

首先我们先要安装aptitude，这是个包管理器，虽然树莓派自带的apt-get也可以安装这些服务器所需，但是会出现一些依赖问题，而aptitude就没有这个问题

在putty中输入：

sudo apt-get update
sudo apt-get update --fix-missing
sudo apt-get install aptitude

然后等待其安装完成，期间可能跳出需要你确认的选项，输入y同意即可

接下来就是用aptitude安装服务器所需的组件了，这里采用的是LNMP的搭配，在putty中输入：

sudo aptitude install nginx php7.3 php7.3-fpm php7.3-mysql php7.3-common mariadb-server

此处若是报错，就是这些东西版本更新了，自行搜索最新版本的名称进行下载，可以输入sudo apt-cache search packagename来搜索最新版本号

之后我们需要对我们的Nginx进行一些简单的配置，输入下列命令：

sudo nano /etc/nginx/sites-available/default

找到和下面相对应的部分并用下面的将原部分覆盖：

# Default server configuration
#
server {
        listen 80 default_server;
        listen [::]:80 default_server;

        root /var/www/html;

        # Add index.php to the list if you are using PHP
        index index.html index.htm index.nginx-debian.html index.php;

        server_name _;

        location / {
                 # First attempt to serve request as file, then
                 # as directory, then fall back to displaying a 404.
                 try_files $uri $uri/ =404;
        }

        # pass PHP scripts to FastCGI server
        #
        location ~ \.php$ {
                 # include snippets/fastcgi-php.conf;
                 #
                 # # With php-fpm (or other unix sockets):
                 fastcgi_pass unix:/run/php/php7.3-fpm.sock;
                 # # With php-cgi (or other tcp sockets):
                 # fastcgi_pass 127.0.0.1:9000;
                 # 设置脚本文件请求的路径
                 fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
                 # 引入fastcgi的配置文件 
                 include fastcgi_params;
        }

}

在putty中输入下列命令重启服务：

sudo /etc/init.d/nginx restart
sudo /etc/init.d/php7.3-fpm restart
sudo service mariadb restart    
sudo service nginx restart

接着输入下列命令授予权限：

sudo chmod 777 /var/
sudo chmod 777 /var/www
sudo chmod 777 /var/www/html

接下来还需要安装phpMyAdmin（这里若是安装失败请自行讲aptitude换成apt-get）：

sudo aptitude install phpmyadmin

其中会跳出几个对话框，第一个对话框在apache2那里按一下空格然后回车，另外几个按自己需求配置

安装完成后还需要把phpmyadmin连接到nginx服务器的php根目录

sudo ln -s /usr/share/phpmyadmin /var/www/html

完成后在浏览器内输入你的树莓派ip/phpmyadmin，能访问则说明配置成功

但是这时我们没有账号来登录，我们要给它设置权限，继续回到putty输入：

sudo mysql -u root -p

这里会提示你输入密码，但是第一次其实没有密码，所以直接按回车即可

然后我们要创建账户，输入下列命令（记得这里有分号）：

CREATE USER 'name'@'localhost' IDENTIFIED BY 'password';

密码和账户就输入你要的

然后执行：

grant all privileges on *.* to 'name'@localhost;
flush privileges;

这里的目的是授予最高权限，name依然填写自己的用户名，成功后输入exit退出

若是不幸多添加了账户，可以输入如下指令来删除：

DROP USER 'name'@'localhost';

然后在浏览器输入你的树莓派ip/phpmyadmin就可以用这里设置的账号登录了

设置博客环境与博客安装

现在我们的流程已经走了一半了，让我们继续

所需系统：

  • wordpress

接下来我们是新建数据库

如图，在数据库中设置，数据库名称随意，编码格式请选择正确，点击创建即可

然后利用VNC的文件传输功能（在VNC访问成功后的顶部）将wordpress传输至树莓派（若是用的其它服务器则有各种传输软件诸如FXP可以传输文件至服务器）

在树莓派中剪切到/var/www/html目录下，然后解压，重命名为自己想要的名称

回到你的浏览器中，输入你的树莓派ip/那个重命名的解压文件夹的名称即可访问wordpress设置界面

数据库名就输入你之前设置的数据库，用户名和密码输入phpmyadmin的用户名和密码，下面的不用管

然后跟着要求配置即可，这里不再过多赘述
